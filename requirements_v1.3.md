# 要件定義書：QRコード影投影STL変換Webアプリ

**ドキュメントバージョン:** 1.3  
**対象:** AIコーディングエージェントによる実装

---

## 1. プロダクト概要

### 目的

ユーザーが任意のSTLファイルとQRコード文字列を入力すると、「**70〜80°の鋭角から点光源を当てたときに壁面にQRコードの影が映る**」よう元のSTL形状を最小限に改変した新しいSTLファイルを出力するWebアプリ。

### 具体的なユースケースイメージ

```
💡スマホライト
  \  ← 70〜80°（ほぼ真横に近い鋭角）
   \
    \
[3Dオブジェクト]
              \
               \
                \
           ┃壁┃QRコードの影┃
```

スマホをほぼ真横に近い角度（光源と壁面のなす角70〜80°）から3Dオブジェクトに当てると、正面の壁にQRコードが映る。影は縦方向に大きく引き伸ばされるが、QRリーダーで正しく読み取れることを保証する。

## 2. ユーザーストーリー

| # | ストーリー |

| U8 | ユーザーはシミュレーター上でQRコードが実際に読み取れるか自動検証できる |

## 3. 機能要件

### 3-1. 入力

| ID | 機能 | 詳細 |
|----|------|------|
| F01 | STLアップロード | バイナリ・ASCIIどちらのSTLも受け付ける。上限100MB |
| F02 | STL姿勢調整 | アップロード後、ビューア上でSTLの向きを調整できる。詳細はF21参照 |
| F03 | QRコード文字列入力 | URL・テキスト・任意文字列。最大2953バイト（QR Version40相当） |
| F04 | 投影パラメータ設定 | 下記パラメータをUIで指定可能にする |

**投影パラメータ一覧**

| パラメータ名 | 説明 | デフォルト値 | 備考 |
|------------|------|------------|------|
| 光源位置 X | STL原点からの光源X座標（mm） | 0 | |
| 光源位置 Y | STL原点からの光源Y座標（mm） | 50 | 70〜80°相当の高さ |
| 光源位置 Z | STL原点からの光源Z座標（mm） | 150 | 70〜80°相当の奥行き |
| 壁面の向き | 壁の法線ベクトル（X, Y, Z） | (0, 0, -1) | 正面壁 |
| 壁面オフセット | 原点から壁面までの距離（mm） | 500 | |
| 投影QRサイズ | 壁面上でのQRコードの1辺サイズ（mm） | 100 | 引き伸ばし後の実寸 |
| QRエラー訂正レベル | L / M / Q / H | **H** | 高歪み条件のためH推奨・デフォルト |
| ボクセル解像度 | 1辺のボクセルサイズ（mm） | 0.5 | |
| 照射角度（表示のみ） | 光源・壁面から自動計算した照射角度 | 読み取り専用 | UIに角度を表示するのみ |
| STL回転行列 | 入力STLビューアで設定した回転をバックエンドに渡す4×4変換行列 | 単位行列 | F21の操作結果を反映 |

> **70〜80°の定義：** 光源方向ベクトルと壁面法線ベクトルのなす角が70〜80°であることを指す。光源Y=50, Z=150 はこの条件を満たす（arctan(50/150) ≈ 18°、壁面法線との角度 ≈ 72°）。

### 3-2. 処理コアロジック（バックエンド）

| ID | 機能 | 詳細 |
|----|------|------|
| F10 | QRコード生成 | 文字列からQRコード画像（白黒2値ビットマップ）を生成する。エラー訂正レベルH固定推奨 |
| F11 | STLボクセル化 | 入力STLにF21で確定した回転行列を適用したうえでボクセルグリッドに変換する |
| F12 | 壁面座標系の構築 | 光源位置・壁面法線から「壁面上のUV座標系」を定義する |
| F13 | QRコード逆透視変換（高歪み対応） | 70〜80°の鋭角投影で壁面上のQRが正方形に見えるよう、QRビットマップをHomography逆行列で変換する。歪み率が大きいため**サブピクセル精度での補間処理**を行う |
| F14 | 逆投影レイキャスト（削り） | 光源位置から壁面「白画素」へ向かう光線上のボクセルをすべて空洞化する |
| F15 | 影保証チェック（黒画素） | 光源位置から壁面「黒画素」へ向かう光線上に最低1ボクセルの素材を保証する |
| F16 | ボクセル→STLメッシュ変換 | 処理済みボクセルグリッドをMarchingCubesでSTLメッシュに変換する |
| F17 | 出力STLエクスポート | バイナリSTLとして出力する |
| F18 | QRコード読み取り自動検証（必須） | シミュレーター上でレンダリングした影画像をQRデコードライブラリに通し、元の文字列と一致するか検証する。不一致の場合は警告を表示しパラメータ調整を促す |

**処理フロー（v1.3）**

```
入力STL
  ↓ F21（ビューア上で回転・姿勢確定）
  ↓ F11（回転行列を適用してボクセル化）
ボクセルグリッド（全セル=1で初期化）

QRコード文字列
  ↓ F10
QRコードビットマップ（白黒2値・エラー訂正H）
  ↓ F12
光源(X,Y,Z) + 壁面法線 → 壁面UV座標系を定義
  ↓ F13
70〜80°高歪み逆透視変換（Homography逆行列＋サブピクセル補間）
→ ボクセル空間にQRをマッピング
  ↓ F14
白画素 → 光源から光線を飛ばし通過ボクセルを0に（削り）
  ↓ F15
黒画素 → 光線上に1ボクセル以上残す（影の存在保証）
  ↓ F16
Marching Cubesでメッシュ再構築
  ↓ F17
出力STL

  ↓ F18
影シミュレーター画像をQRデコード → 元文字列と照合
  ✓ 一致 → ダウンロード許可
  ✗ 不一致 → 警告表示・パラメータ調整を促す
```

#### F12・F13の数理補足（実装者向け）

**壁面UV座標系の構築（F12）**

壁面法線ベクトル **n** に対して、壁面上の基底ベクトル **u**, **v** をグラムシュミット法で構築する。

**逆透視変換（F13）**

光源を **L**、壁面上の点を **P** とすると光線は以下で表される。

```
R(t) = L + t * (P - L)   (0 ≤ t ≤ 1)
```

70〜80°の鋭角では投影影の縦方向引き伸ばし率が以下のように大きくなる。

```
引き伸ばし率 = 1 / cos(θ)
θ=70° → 約2.9倍
θ=80° → 約5.8倍
```

QRビットマップの各画素をHomography逆行列で変換する際、**バイリニア補間またはランチョス補間**を用いてサブピクセル精度を確保する。補間なしでは黒白境界がジャギーになりQRの読み取り精度が著しく低下する。

**QRモジュールの最小ボクセル数の制約**

```
必要ボクセル数 = ceil(投影QRサイズ / (QRモジュール数 × ボクセル解像度)) ≥ 3
```

壁面上で1モジュールが最低3px以上になるようQRサイズとボクセル解像度を自動調整する。

### 3-3. プレビュー・UI

| ID | 機能 | 詳細 |
|----|------|------|
| F20 | 3Dビューア（入力STL） | アップロードしたSTLをThree.jsで表示。回転ギズモ・軸別回転ボタン・光源ギズモを表示する |
| F21 | STL回転操作（入力STLビューアのみ） | 以下の2種類の操作UIを提供する |
| F22 | 3Dビューア（出力STL） | 変換後STLをThree.jsで表示・自由回転 |
| F23 | 影シミュレーター | 70〜80°投影時の縦方向引き伸ばしを正確に再現してQR影をプレビュー |
| F24 | 進捗表示 | ボクセル化→逆透視変換→レイキャスト→メッシュ化の各ステップの進捗バーを表示 |
| F25 | QR自動検証表示 | F18の結果を「✓ QR読み取り成功」または「✗ 読み取り失敗：パラメータを調整してください」としてUIに表示 |
| F26 | ダウンロードボタン | F18の検証が成功した場合のみ有効化する |

**F21 STL回転操作の詳細**

**① 回転ギズモ（ドラッグ操作）**  
Three.jsの `TransformControls`（rotateモード）を使用。X・Y・Z軸それぞれのリングをドラッグして任意角度に回転できる。

**② 軸別 ±90° 回転ボタン**  
ビューア下部に以下6ボタンを設置する。

```
[ X +90° ]  [ Y +90° ]  [ Z +90° ]
[ X -90° ]  [ Y -90° ]  [ Z -90° ]
```

ボタンを押すと対応軸に90°回転し、ビューアに即時反映される。確定した回転は4×4変換行列としてバックエンドに渡し、F11のボクセル化処理に反映する（Three.js上の見た目だけでなく実データに適用）。

## 4. 非機能要件

| 区分 | 要件 |
|------|------|
| パフォーマンス | ボクセル解像度0.5mm・10cm角オブジェクトで5分以内に処理完了 |
| ブラウザ対応 | Chrome / Firefox 最新版 |
| ファイルサイズ | 出力STL 200MB以下 |
| エラーハンドリング | STL破損・QR生成失敗・メモリ超過・光源がオブジェクト内部に位置するケースで日本語エラーメッセージを表示 |
| セキュリティ | アップロードファイルはセッション単位で管理し処理後に削除 |

## 5. 技術スタック（推奨）

| レイヤー | 技術 | 理由 |
|---------|------|------|
| フロントエンド | React + Three.js | STL表示・回転ギズモ・影シミュレーターのため |
| バックエンド | Python（FastAPI） | numpy/scipy/skimageによる数値計算が豊富 |
| QR生成 | `qrcode`ライブラリ（Python） | エラー訂正レベルH対応 |
| QR検証 | `pyzbar` または `opencv-python` QRDecoder | 影画像からのQRデコード |
| STL入出力 | `numpy-stl` | バイナリSTLの読み書き |
| ボクセル化 | `trimesh` | STL→ボクセル変換 |
| 逆透視変換 | `opencv-python` | Homography行列・サブピクセル補間 |
| メッシュ再構築 | `scikit-image` MarchingCubes | ボクセル→STL変換 |
| 非同期処理 | FastAPI BackgroundTasks または Celery + Redis | 重い処理をバックグラウンドで実行 |

## 7. 未解決事項・実装上の注意点

| # | 項目 | 備考 |
|---|------|------|
| T1 | 意匠性最適化はスコープ外（v1） | 形状カモフラージュは将来対応 |
| T2 | ボクセル解像度とメモリのトレードオフ | 0.5mm解像度・10cm角 = 200³ = 800万ボクセル。メモリ上限に注意 |
| T3 | 光源がSTL内部・背後に位置するケースの除外 | 光線方向が逆転するためエラーとして弾く |
| T4 | QRモジュールサイズとボクセル解像度の整合 | 1QRモジュール = 最低3ボクセルになるよう自動スケーリング必須 |
| T5 | 推奨動作範囲の明示 | 照射角度70〜80°を推奨範囲とし、20°未満・85°以上は警告表示 |
| T6 | QRコード読み取り保証は必須 | F18のQR自動検証をパスしない限りダウンロードを許可しない |
| T7 | 80°に近づくほど影が極端に伸びる | θ=80°で引き伸ばし率約5.8倍。壁面オフセットや投影QRサイズの設定ガイドをUIに表示する |
| T8 | エラー訂正レベルHの限界 | エラー訂正Hでも訂正できるのはデータの約30%まで。歪みが極端な場合はQRバージョン（モジュール数）を下げてデータ量を減らすことを検討 |
| T9 | STL回転のバックエンド反映 | F21で確定した4×4回転行列はフロントエンドからバックエンドへJSON形式で渡し、F11のボクセル化前にSTL頂点座標に適用する |
